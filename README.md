# civic-starter

A clean, forkable starter template for the **Civic Design System**, built on the U.S. Web Design System (USWDS). Vanilla HTML/CSS with everything you need to launch an accessible, mobile-friendly site — styles, structure, and build system in one repo. No framework required.

## How it works with civic-guide

**civic-starter** is the companion to [civic-guide](https://github.com/michaelasaro/civic-guide), the full reference site for the Civic Design System. The two repos are designed to work together:

| Repo | Purpose |
|------|---------|
| **civic-guide** | The reference site. Explore components, customize your theme, preview changes in context. |
| **civic-starter** | Your clean starting point. Fork this, copy your customized styles, and build your site. |

### Workflow

1. **Customize your theme** in civic-guide (colors, typography, spacing, components)
2. **Copy your `src/styles/` folder** from civic-guide into this project
3. **Run `npm run build`** to compile your customized theme
4. **Start building your pages** on top of this clean template

## Getting started

```bash
npm install
npm run build
npm run serve
```

Your site will be running at [http://localhost:3001](http://localhost:3001).

## Scripts

| Command | What it does |
|---------|-------------|
| `npm run build` | Clean and build all assets (styles, fonts, images, USWDS) |
| `npm run build:styles` | Compile styles only |
| `npm run watch:styles` | Watch for style changes and recompile automatically |
| `npm run serve` | Start local development server on port 3001 |
| `npm run clean` | Remove the `dist/` folder |

## How the build works

The build system uses Gulp 5 + Dart Sass + PostCSS:

1. **Sass compilation** — `src/styles/index.scss` loads three layers:
   - `_bridge.scss` — connects your settings to USWDS core
   - `_packages.scss` — selects which USWDS components to include
   - `custom-styles.scss` — your custom CSS overrides
2. **Autoprefixer** — adds vendor prefixes for browser compatibility
3. **Output** — `dist/civic/styles/civic.css` + sourcemap
4. **Minification** — `dist/civic/styles/civic.min.css` + sourcemap

The build also copies fonts, images, and stock USWDS assets into `dist/`.

## Customizing your theme

All theme settings live in `src/styles/uswds-settings/`. Each file is organized with clear documentation:

- **`_colors.scss`** — Primary, secondary, and accent colors
- **`_typography.scss`** — Fonts, sizes, weights, line heights
- **`_spacing.scss`** — Border radius, grid containers, site margins
- **`_components.scss`** — Per-component settings (header, footer, cards, etc.)
- **`_utilities.scss`** — Which utility classes to generate
- **`_general.scss`** — Image paths, focus styles, and other general settings

To change a setting:
1. Open the settings file and find the setting you want
2. Uncomment the line (remove the `//`) and set your value
3. Uncomment the matching line in `_bridge.scss`
4. Run `npm run build` (or `npm run watch:styles` to auto-recompile)

## Choosing components

Not every site needs every USWDS component. Open `src/styles/_packages.scss` to see the full list. Comment out packages you don't use to reduce your CSS file size.

## Including styles in your HTML

```html
<!-- CSS (in head) -->
<link rel="stylesheet" href="dist/uswds/css/uswds.min.css">
<link rel="stylesheet" href="dist/civic/styles/civic.css">

<!-- JavaScript (before closing body tag) -->
<script src="dist/uswds/js/uswds.min.js"></script>
```
